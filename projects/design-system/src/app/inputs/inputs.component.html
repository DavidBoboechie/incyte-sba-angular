Inputs

As for the buttons, the inputs are based on Bootstrap's classes that we overwrite.

Text input

Only the input is required and it needs the .form-control class applied.

You can optionally add a label which needs the .form-label class, and within it you can add a span with the .info class for a greyer text.

If you need an error message to be displayed in case of invalid input, you can add right after the input a div with the .invalid-feedback class and containing the desired text within it.

Input with icon

If you have a compatible recent browser, it should work just fine using the basic Bootstrap classes, otherwise you will need to add the .input-icon class with the .input-group one.

The same applies to the input styling in case of disabled/valid/invalid status where it will work with no changes only for the compatible browsers, otherwise the .input-group div will require the classes .disabled, .valid and .invalid.

Textarea

The textarea works exactly as the text input, just apply it the .form-control class, and you can optionally add the label and the error message.

Checkbox / Radio button

The checkbox and the radio button require to be included in a div with the .form-check class, and to themselves have .form-check-input.

Badges

The badges use Bootstrap's .badge class, for which you can choose the color using the usual classes like .bg-primary, .bg-secondary and others. Note that custom coloring is already defined in SBA (see metadata.scss file).

For a smaller badge, you can add the .sq-small-badge class.
